# Components Folder Guide

`components`는 한 화면에 묶이지 않는 공용 UI를 관리합니다. 특정 기능의 업무 로직보다 레이아웃, 검색, 카탈로그, 쇼케이스처럼 여러 곳에서 다시 쓰는 화면 조각을 둡니다.

## 폴더 역할

| 경로 | 역할 |
| --- | --- |
| `common/` | Button, Card, FormField, Section 같은 가장 작은 UI 단위입니다. |
| `catalog/` | 카탈로그 목록, 카드, 검색 입력, 빈 상태를 구성합니다. |
| `header/` | 브랜드 영역, 헤더 액션, 메뉴, 상단 링크를 나눠 관리합니다. |
| `search/` | 전역 검색 모달의 입력, 결과, 최근 검색, 빈 상태를 담당합니다. |
| `showcase/` | 프리뷰 카드, 지표, 활동 목록, 리소스 그리드, 저장된 프리뷰를 관리합니다. |

## 루트 컴포넌트

| 파일 | 역할 |
| --- | --- |
| `Header.tsx` | 상단 헤더를 조립합니다. |
| `Layout.tsx` | 헤더, 사이드바, 본문, 전역 검색을 묶는 앱 레이아웃입니다. |
| `SearchModal.tsx` | 검색 모달의 전체 흐름을 연결합니다. |
| `Sidebar.tsx` | 섹션과 하위 라우트 탐색을 제공합니다. |

## 배치 기준

- 여러 페이지에서 같은 목적과 같은 모양으로 쓰이면 `components/`에 둡니다.
- 버튼, 카드처럼 가장 작은 단위는 `common/`에 둡니다.
- 카탈로그 화면에서만 반복되는 패턴은 `catalog/`에 둡니다.
- 쇼케이스 페이지에서만 반복되는 패턴은 `showcase/`에 둡니다.
- 특정 feature 상태나 저장 로직이 강하게 묶인 UI는 `features/`에 둡니다.
- 브라우저 저장소 key, 검색 기록, UI helper는 해당 하위 폴더 안에서 가까이 관리합니다.
